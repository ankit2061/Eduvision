01Foundation

Vision & Problem Statement

Every classroom in India has students with varied learning abilities ‚Äî from visual learners and auditory learners to students with blindness, deafness, autism, dyslexia, stammering, ADHD, and physical disabilities. Yet all content is prepared in one format, designed for one kind of student.

Teachers who want to create inclusive content for all their students spend an additional 8‚Äì10 hours per week manually converting lessons. Given their already packed schedules, this is simply not sustainable. The result: students with disabilities are left behind, not because teachers don't care, but because the system doesn't help.

EduVoice solves this by being the invisible bridge between any content a teacher creates and any format a student needs. The teacher creates once. The platform delivers to everyone.

04Teacher Features

Teacher (Normally Abled) ‚Äî Full Feature Set

A normally abled teacher interacts with EduVoice through a standard interface. Their primary job is to create content in any form they prefer ‚Äî and EduVoice does the heavy lifting of adapting it for every student in the class.

Content Creation ‚Äî Written Material

1
Teacher types or pastes written content

Could be a lesson explanation, a chapter summary, a concept note, or any text-based material. Supports markdown and rich text formatting.

2
Teacher selects target class

The class roster (with each student's disability/learning profile) is already loaded from the admin. EduVoice knows exactly what formats are needed.

3
AI Conversion Engine runs automatically

Gemini processes the content and generates: simplified version (for low-comprehension students), grade-level version, advanced version, audio narration (ElevenLabs), large-print version, dyslexia-friendly formatted version, high-contrast version, and a sign language keyword map for visually supported reading.

4
Teacher previews and publishes

Teacher sees a preview of the content across formats. Can make edits before publishing. One click publishes to the entire class ‚Äî each student automatically receives their adapted version.

Content Creation ‚Äî Audio Material

If the teacher prefers to speak instead of type, EduVoice supports audio-first content creation equally.

1
Teacher records or uploads an audio file

Can record live in the app using a microphone, or upload a pre-recorded .mp3/.wav file. This is especially useful for language classes or explanation-heavy topics.

2
Audio is transcribed by Gemini

The audio is converted to text using speech-to-text. This transcript becomes the master content for all further conversions.

3
Same conversion engine as written material runs

All formats are generated identically to the written material flow. For hearing-impaired students, the transcript itself becomes the primary content, with captions and sign keyword overlays added.

Content Creation ‚Äî Video Material

1
Teacher uploads a video or records screen + webcam

EduVoice accepts video uploads. The app can also generate a simple screen recording interface for teacher explainer videos.

2
Video is processed for accessibility

Auto-captions are generated and overlaid for hearing-impaired students. The audio track is extracted and transcribed. The transcript is converted to all written formats. A sign language keyword panel is generated alongside the video player for deaf students.

3
Blind/low-vision students get an audio-described version

Gemini generates a rich audio description of any visual elements in the video. ElevenLabs narrates it. This is served alongside the video as an "audio description track."

06Core Engine

Content Conversion Engine

This is the heart of EduVoice. When a teacher publishes content, the conversion engine runs in the background and produces a personalised output for every student profile in the class.

Master Conversion Pipeline ‚Äî Runs Once Per Lesson Published
1
Input: Raw Content from Teacher

Written text, audio recording, or video ‚Äî in any combination. The engine accepts all three and normalises them to a unified text transcript first.

2
Class Profile Analysis

The engine reads the disability and learning style profile of every student in the class from Snowflake. It determines the unique set of output formats needed. For example: if the class has 2 blind students, 1 deaf student, 3 dyslexic students, and 5 visual learners ‚Äî it queues exactly those output formats and no others.

3
Parallel Generation via Gemini

Gemini generates all text-based variants simultaneously: simplified reading level, grade-level, advanced level, dyslexia-reformatted layout, structured summary for autistic students (clear bullet points, no ambiguous language), visual description layer for blind students, simplified vocabulary version for intellectual disabilities.

4
Audio Generation via ElevenLabs

For every student who needs audio: a narrated version is generated at their preferred speaking speed. Slower pace for students who need more processing time. The blind student gets a full audio description version including descriptions of any diagrams or images referenced.

5
Sign Language Keyword Map

Key vocabulary words in the lesson are extracted and mapped to Indian Sign Language (ISL) GIF/video clips. These appear as hover overlays for deaf students when they tap a highlighted word while reading.

6
Storage in Snowflake + DigitalOcean Spaces

All generated variants are stored. Text and metadata go into Snowflake VARIANT columns. Audio and video files are stored in DigitalOcean Spaces. Each student's profile is linked to their specific variant ‚Äî so when they open the lesson, they always get their version.

7
Delivery to Student

When the student opens the lesson, the app fetches their personalised variant. No filtering or selection needed from the student's side ‚Äî it just works. They see exactly the right format for them.

08Assessment

Tests & Assignments ‚Äî Adaptive System

Tests and assignments are one of the most critical areas for inclusion. The same knowledge must be assessed ‚Äî but the format of the assessment must not disadvantage a student due to their disability. EduVoice automates this completely.

How It Works ‚Äî Teacher Side

1
Teacher creates a standard test

Teacher writes questions normally ‚Äî multiple choice, short answer, long answer, match the following, or fill in the blanks. This is the "master test."

2
AI generates adapted variants for every disability profile in the class

This happens automatically. The teacher doesn't need to do anything extra. The engine produces as many variants as there are distinct profiles in the class.

3
Teacher sets the due date and publishes

Each student automatically gets their adapted version when they open the test. Same deadline, same knowledge assessed, different format.

4
Teacher reviews all submissions in a unified view

The teacher sees all answers in a standardised format regardless of how each student submitted (voice, typed, MCQ, drawing). Gemini normalises and evaluates, presenting the teacher with scores and the original answer for review.

12AI Engineering

Gemini Prompt Templates

These are the exact prompts to use. Store them as constants in utils/prompts.py. Use Python f-strings or .format() to inject variables.

1. Differentiated Lesson Generation

Gemini Prompt """ You are an expert Indian school curriculum writer aligned with NEP 2020. Create a differentiated reading lesson on: {topic} for Grade {grade}. Generate exactly {tiers} versions labelled: Foundation, Grade-Level, Advanced. For EACH version provide: 1. A {word_count}-word passage (appropriate vocabulary for that level) 2. 3 comprehension questions with model answers 3. 3 key vocabulary words with simple, one-sentence definitions Additional rules: - Language: {language} (default: English) - No idioms or culturally ambiguous phrases - Use Indian examples, names, and contexts where relevant - Keep sentences short at Foundation level (max 12 words per sentence) Return ONLY a valid JSON object. No preamble, no explanation. Schema: { "tiers": [ { "level": 1, "label": "...", "text": "...", "questions": [{"q": "...", "a": "..."}], "vocabulary": [{"word": "...", "definition": "..."}] } ] } """

2. Autism-Friendly Content Reformatter

Gemini Prompt """ You are rewriting educational content for a student with Autism Spectrum Disorder. Original content: {content} Rewrite rules (follow ALL of them): - Use only literal, direct language. Remove ALL idioms, metaphors, and figurative language. - Replace every idiom with its literal meaning. (e.g., "It's raining cats and dogs" ‚Üí "It is raining very heavily.") - Use short sentences (max 10 words each). - Use bullet points for lists. Never use flowing prose for listing things. - Add a "What this means" sentence in plain language after any complex concept. - Do NOT use: rhetorical questions, sarcasm, humour, or ambiguous phrasing. - Structure: Title ‚Üí Objective (one sentence: "By the end of this, you will know...") ‚Üí Content ‚Üí Summary bullet points. - Tone: calm, neutral, matter-of-fact. Return ONLY the reformatted text. No explanation or commentary. """

3. Stammer-Friendly Speech Analysis

Gemini Prompt """ You are a compassionate English speech coach working with Indian students. Student transcript: "{transcript}" Reference text: "{reference_text}" Stammer-friendly mode: {stammer_friendly} Analysis rules: - If stammer_friendly is true: IGNORE all word repetitions (e.g. "I I I want"), long pauses (marked as "..."), and partial words (e.g. "I wa- want"). Reconstruct the intended sentence from completed words only. Key Analytics Query (for Demo)

This single query demonstrates why Snowflake's VARIANT type matters ‚Äî it joins structured scores with unstructured JSON accessibility settings to prove impact:

Snowflake SQL SELECT ps.accessibility_mode_used:dyslexia_font::BOOLEAN AS dyslexia_font_active, ps.accessibility_mode_used:stammer_friendly::BOOLEAN AS stammer_mode_active, COUNT(*) AS sessions, ROUND(AVG(ps.grammar_score), 1) AS avg_grammar_score, ROUND(AVG(ps.pronunciation_score), 1) AS avg_pronunciation_score FROM practice_sessions ps JOIN users u ON ps.student_id = u.user_id WHERE u.institution_id = 'SCHOOL_ID' GROUP BY 1, 2 ORDER BY dyslexia_font_active DESC; -- Result example: -- dyslexia_font_active | stammer_mode_active | sessions | avg_grammar | avg_pronunciation -- TRUE | FALSE | 47 | 78.3 | 74.1 -- FALSE | FALSE | 203 | 66.9 | 68.5 -- This proves dyslexia font is associated with a 12-point grammar improvement.
